# Playbook: إصلاح خطأ (Bug Fix)

## الخطوات

### 1. Think (مختصر)

- [ ] أعد إنتاج الخطأ
- [ ] حدد السبب الجذري (root cause)
- [ ] قيّم التأثير (كم مستخدم؟ هل أمني؟)

### 2. Plan (مختصر)

- [ ] سجّل في [.tasks/Bugs.md](../../.tasks/Bugs.md)
- [ ] حدد الإصلاح المقترح
- [ ] اكتب اختبار regression

### 3. Build

- [ ] أصلح الخطأ
- [ ] أضف اختبار regression
- [ ] commit + push

### 4. Review

- [ ] Security review (إن كان أمنيًا)
- [ ] QA: اختبار regression
- [ ] Documentation (إن تغير سلوك موثق)

### تصنيف الأخطاء

| الخطورة | المراجعة المطلوبة | وقت الاستجابة |
|---|---|---|
| حرج (أمني/بيانات) | Security + QA + Architect | فوري |
| عالي (وظيفي) | QA | نفس السبرنت |
| متوسط | QA | السبرنت التالي |
| منخفض | اختياري | backlog |
