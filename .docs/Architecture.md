## الطبقات

### 1. Client Layer
- **Web App**: React/Next.js — واجهة المحادثة والبحث
- **Mobile**: React Native (مرحلة لاحقة)
- **API SDK**: مكتبة للتكامل الخارجي

### 2. API Gateway
- مصادقة JWT / OAuth2
- Rate limiting حسب الخطة
- توجيه الطلبات للخدمات

### 3. الخدمات (Services)

| الخدمة | المسؤولية |
|---|---|
| **Chat Service** | إدارة المحادثات، streaming، سياق الجلسة |
| **Doc Service** | رفع، فهرسة، واسترجاع الوثائق القانونية |
| **User Service** | حسابات، صلاحيات، اشتراكات |

### 4. طبقة الذكاء الاصطناعي
- **LLM**: نموذج لغوي (قابل للتبديل — OpenAI, Anthropic, local)
- **Embedder**: تحويل النصوص لمتجهات
- **Retriever**: RAG — استرجاع المقاطع ذات الصلة قبل الإجابة

### 5. طبقة البيانات
- **PostgreSQL**: بيانات المستخدمين، المحادثات، البيانات الوصفية
- **Vector DB**: فهرسة الوثائق القانونية (pgvector أو Qdrant)
- **Redis**: cache، جلسات، قوائم انتظار

## Engineering OS (الطبقة الافتراضية)

فوق كل هذا، يعمل **Adalah Engineering OS**:

```
.docs/     → المعرفة الثابتة (وثائق، ADR، API)
.ai/        → العمليات (وكلاء، مهارات، playbooks)
.tasks/     → العمل الجاري (سبرنتات، مهام)
.cursor/    → قواعد التنفيذ لـ Cursor
```

هذه الطبقة **ليست كود تشغيل** — بل هي نظام التشغيل الذي يوجّه كل تطوير.

## مبادئ معمارية

1. **Separation of Concerns** — كل خدمة لها مسؤولية واحدة
2. **API-First** — كل ميزة تبدأ بتصميم API
3. **Event-Driven** حيث يلزم — فهرسة الوثائق، إشعارات
4. **Pluggable AI** — LLM قابل للتبديل دون تغيير باقي النظام
5. **Security by Default** — تشفير، RBAC، audit logs
