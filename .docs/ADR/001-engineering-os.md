# ADR-001: اعتماد Adalah Engineering OS

**الحالة**: مقبول  
**التاريخ**: 2026-07-05  
**صاحب القرار**: Chief Architect  
**يحل محل**: —

## السياق

مشروع عدالة منصة طويلة الأمد. الاعتماد على محادثات طويلة مع نماذج الذكاء الاصطناعي يؤدي إلى:

- فقدان السياق بين الجلسات
- تكرار القرارات الهندسية
- عدم اتساق جودة الكود
- صعوبة الانتقال بين أدوات AI مختلفة

## الخيارات المدروسة

### الخيار 1: Prompts طويلة فقط
- **المزايا**: بسيط، لا يحتاج هيكل ملفات
- **التنازلات**: غير مستدام، لا يُنقل بين الأدوات، يفقد السياق

### الخيار 2: .cursor/rules تقليدية فقط
- **المزايا**: يعمل مع Cursor مباشرة
- **التنازلات**: مرتبط بأداة واحدة، لا يغطي التخطيط والمراجعة

### الخيار 3: Engineering OS كامل (المختار)
- **المزايا**: معرفة في المشروع، قابل للنقل، فريق افتراضي، سير عمل موحد
- **التنازلات**: جهد تأسيس أولي، يحتاج صيانة الوثائق

## القرار

اعتماد **Engineering OS** بهيكل:

```
.docs/   → Knowledge Base (100+ وثيقة مستهدفة)
.ai/     → Agents, Skills, Playbooks, Workflows
.tasks/  → Sprints, Bugs, Features
.cursor/ → قواعد تنفيذ Cursor
```

مع سير عمل رباعي: **Think → Plan → Build → Review**

وفريق افتراضي من 8 وكلاء متخصصين.

## العواقب

- كل ميزة جديدة تمر بالسير الكامل قبل الدمج
- الوثائق جزء من تعريف "تم" (Definition of Done)
- Cursor محرك تنفيذ فقط — العقل الحقيقي في ملفات المشروع
- يمكن الانتقال لأي أداة AI دون إعادة بناء العمليات
