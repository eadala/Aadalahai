# Skill: تحليل المتطلبات

## متى تُستخدم

مرحلة **Think** — قبل أي تصميم أو كود.

## الخطوات

1. **اقرأ الطلب** — ما الميزة المطلوبة بالضبط؟
2. **حدد المستخدم** — من سيستخدمها؟ (محامي، مواطن، مطور)
3. **استخرج المتطلبات الوظيفية** — ماذا يجب أن تفعل؟
4. **استخرج المتطلبات غير الوظيفية** — أداء، أمان، قابلية توسع
5. **حدد القيود** — ما الموجود مسبقًا؟ ما التبعيات؟
6. **اسأل** — ما الغامض؟ اطرح أسئلة قبل المتابعة

## المخرج

```markdown
## تحليل المتطلبات: [اسم الميزة]

### المستخدم المستهدف
...

### المتطلبات الوظيفية
- [ ] ...

### المتطلبات غير الوظيفية
- ...

### القيود
- ...

### أسئلة مفتوحة
- ...
```
