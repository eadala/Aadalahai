# Skill: مقارنة التصاميم

## متى تُستخدم

مرحلة **Think** — بعد تحليل المتطلبات.

## الخطوات

1. **اقترح 2-3 تصاميم** مختلفة على الأقل
2. **قيّم كل تصميم** على المحاور:
   - البساطة
   - الأداء
   - الأمان
   - قابلية التوسع
   - سرعة التنفيذ
   - قابلية الصيانة
3. **وضّح المزايا والتنازلات** لكل خيار
4. **اختر** مع تبرير واضح
5. **حدد المخاطر** للخيار المختار

## المخرج

```markdown
## مقارنة التصاميم: [اسم الميزة]

### الخيار 1: [الاسم]
| المحور | التقييم (1-5) | ملاحظات |
|---|---|---|
| البساطة | | |
| الأداء | | |
| ...

**المزايا**: ...
**التنازلات**: ...

### الخيار 2: [الاسم]
...

### القرار: الخيار X
**السبب**: ...

### المخاطر
| المخاطرة | الاحتمال | التأثير | التخفيف |
|---|---|---|---|
| ... | | | |
```
